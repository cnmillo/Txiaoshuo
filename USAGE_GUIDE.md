# 全自动小说生成器使用指南

本文档提供了全自动小说生成器的详细使用指南，帮助您快速上手并充分利用系统的各项功能。

## 1. 系统登录与初始化

### 1.1 访问系统

- **开发环境**：http://localhost:5173
- **生产环境**：http://localhost（Docker部署）或您配置的域名

### 1.2 配置AI模型

首次使用前，您需要配置AI模型：

1. 点击顶部导航栏的"设置"按钮
2. 在"AI模型配置"部分，选择以下方式之一：
   - **使用OpenAI API**：输入您的OpenAI API密钥
   - **使用自定义CURL**：输入完整的CURL命令，用于调用其他AI模型
3. 点击"保存设置"按钮

## 2. 小说创作流程

### 2.1 创建新小说

1. 在首页点击"创建小说"按钮
2. 填写以下信息：
   - **小说标题**：输入小说的标题
   - **创作提示**：输入小说的核心概念或开头
   - **小说风格**：选择适合的风格（玄幻、武侠、言情、科幻、悬疑等）
   - **目标字数**：设置小说的目标总字数
   - **章节数**：设置小说的章节数量
3. 点击"创建小说"按钮

### 2.2 生成小说内容

1. 进入小说详情页面
2. 点击"开始生成"按钮
3. 系统会自动生成小说内容，包括：
   - 小说大纲
   - 各章节内容
   - 人物设定
4. 生成过程中，您可以：
   - 查看生成进度
   - 暂停/继续生成
   - 调整生成参数

### 2.3 编辑小说内容

1. 在小说详情页面，点击"编辑"按钮
2. 使用富文本编辑器修改内容：
   - 编辑章节内容
   - 添加/删除章节
   - 调整章节顺序
3. 点击"保存"按钮保存修改

### 2.4 导出小说

1. 在小说详情页面，点击"导出"按钮
2. 选择导出格式：
   - **TXT**：纯文本格式，适合一般阅读
   - **PDF**：便携式文档格式，适合打印
   - **EPUB**：电子书格式，适合电子书阅读器
3. 点击"导出"按钮，系统会生成并下载文件

## 3. 高级功能使用

### 3.1 大纲管理

1. 在顶部导航栏点击"大纲管理"按钮
2. 功能包括：
   - **创建大纲**：输入大纲标题和内容
   - **编辑大纲**：修改现有大纲
   - **按大纲生成**：根据大纲生成小说内容
   - **大纲模板**：使用预设的大纲模板

### 3.2 人物管理

1. 在顶部导航栏点击"人物管理"按钮
2. 功能包括：
   - **创建人物**：输入人物姓名、性别、年龄、背景等信息
   - **编辑人物**：修改人物信息
   - **人物关系**：设置人物之间的关系
   - **人物设定**：详细描述人物的性格、外貌、技能等

### 3.3 风格设置

1. 在顶部导航栏点击"风格设置"按钮
2. 功能包括：
   - **选择风格**：从预设风格中选择
   - **自定义风格**：设置语言风格、叙事方式、情感基调
   - **风格模板**：保存和加载风格模板

### 3.4 一致性检查

1. 在顶部导航栏点击"一致性管理"按钮
2. 功能包括：
   - **自动检查**：系统自动检测小说中的前后矛盾
   - **手动检查**：选择特定章节进行检查
   - **检查结果**：查看检查结果并进行修改

### 3.5 版本控制

1. 在小说详情页面，点击"版本管理"按钮
2. 功能包括：
   - **查看历史版本**：查看小说的历史修改记录
   - **恢复版本**：恢复到之前的版本
   - **比较版本**：比较不同版本之间的差异

## 4. AI模型配置

### 4.1 OpenAI API配置

1. 访问OpenAI官网（https://platform.openai.com/）
2. 创建API密钥
3. 在系统设置页面，输入API密钥
4. 选择合适的模型（如gpt-4、gpt-3.5-turbo等）

### 4.2 自定义CURL配置

如果您想使用其他AI模型，可以使用自定义CURL命令：

```bash
curl -X POST https://api.example.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "model": "model-name",
    "messages": [
      {"role": "system", "content": "你是一位专业的小说作家"},
      {"role": "user", "content": "请根据以下提示生成小说内容：{{prompt}}"}
    ],
    "temperature": 0.7,
    "max_tokens": 2000
  }'
```

- `{{prompt}}`：系统会自动替换为实际的创作提示
- 确保CURL命令能够正确返回JSON格式的响应，包含生成的内容

## 5. 最佳实践

### 5.1 创作提示编写

编写有效的创作提示：

- **具体明确**：提供详细的背景信息和要求
- **设定场景**：描述具体的场景和环境
- **人物设定**：提供主要人物的基本信息
- **风格要求**：明确小说的风格和基调
- **长度要求**：指定生成内容的长度

**示例提示**：
> 请创作一部玄幻小说，讲述一个普通少年意外获得上古传承，踏上修行之路的故事。要求：风格大气磅礴，人物性格鲜明，情节跌宕起伏，包含修炼、战斗、友情等元素。

### 5.2 生成参数调整

- **温度**：控制生成内容的随机性，建议设置为0.7-0.9
- **最大 tokens**：控制单次生成的长度，建议设置为1000-3000
- **生成速度**：根据网络状况和AI模型响应速度调整

### 5.3 内容优化

- **多次生成**：对于重要章节，可以多次生成并选择最佳版本
- **人工编辑**：生成后进行人工编辑，调整情节和语言
- **一致性检查**：使用系统的一致性检查功能确保内容连贯

## 6. 常见问题与解决方案

### 6.1 AI生成失败

- **问题**：生成过程中出现错误
- **解决方案**：
  - 检查API密钥是否正确
  - 确保网络连接正常
  - 尝试调整生成参数，减少单次生成的字数
  - 检查AI模型的使用限制

### 6.2 导出失败

- **问题**：导出小说时出现错误
- **解决方案**：
  - 确保小说内容不为空
  - 检查磁盘空间是否充足
  - 尝试使用不同的导出格式
  - 检查文件权限

### 6.3 系统响应缓慢

- **问题**：系统响应速度慢
- **解决方案**：
  - 关闭不必要的浏览器标签页
  - 清理浏览器缓存
  - 检查网络连接速度
  - 考虑使用性能更好的AI模型

### 6.4 内容质量问题

- **问题**：生成的内容质量不高
- **解决方案**：
  - 优化创作提示，提供更详细的要求
  - 调整AI模型参数
  - 使用风格设置功能
  - 进行人工编辑和优化

## 7. 性能优化建议

1. **硬件要求**：
   - 服务器：至少2GB内存，1CPU核心
   - 客户端：现代浏览器，稳定的网络连接

2. **软件优化**：
   - 使用最新版本的Node.js和npm
   - 生产环境使用PM2进行进程管理
   - 配置适当的内存限制

3. **网络优化**：
   - 使用CDN加速静态资源
   - 启用Gzip压缩
   - 优化API请求，减少不必要的网络调用

## 8. 安全注意事项

1. **API密钥保护**：
   - 不要在代码中硬编码API密钥
   - 使用环境变量或配置文件管理敏感信息
   - 定期更换API密钥

2. **数据安全**：
   - 定期备份小说数据
   - 注意保护个人隐私信息
   - 避免在小说中包含敏感信息

3. **使用规范**：
   - 遵守相关法律法规
   - 尊重知识产权
   - 合理使用AI模型，避免滥用
